1 · The paper

Question and contribution

Question: how connected are the world's major exchange rates, and how does that connectedness move over time? More fundamentally: can we measure dynamic connectedness without the rolling windows that Diebold and Yilmaz (2009, 2012, 2014) relied on?

Contribution. Replace the rolling-window VAR inside the Diebold-Yilmaz machinery with a time-varying parameter VAR (TVP-VAR) estimated by a Kalman filter with forgetting factors (Koop and Korobilis 2013, 2014) and a time-varying variance-covariance matrix. The connectedness measures are then recomputed every period from the filtered parameters.

The paper improves the Diebold-Yilmaz method in four ways: (i) parameter change is captured more accurately, (ii) outliers do not distort the estimates, (iii) no arbitrary rolling-window size, and (iv) no loss of observations.

Evidence for the claims: Monte Carlo (Section 3, Table 1)

The authors simulate a VAR(1) 10,000 times (T = 400) and compare how far the estimated time-varying parameters sit from the truth, for a 200-observation rolling window vs the TVP-VAR:

$$DV_t = D_{RW,t}-D_{TVP,t}, \qquad D_{\cdot,t}=|\hat A_{\cdot,t}-A|$$
ScenarioDV11DV12DV21DV22
Single outlier0.025***0.016***0.012***0.027***
Structural break in parameters0.033***0.040***0.032***0.028***

All deviations are significantly positive: the rolling-window estimates sit further from the true parameters than the TVP-VAR estimates, both when a single outlier hits and when the parameters break. This is the paper's evidence for improvements (i) and (ii); improvements (iii) and (iv) hold by construction.

Data used in the paper (Section 4, Table 2)

  • Levels are non-stationary (ERS test), so the paper uses first log differences: \(y_t=\ln x_t - \ln x_{t-1}\); all return series are stationary
  • Return variances (Table 2): GBP 5.988 < EUR 5.996 < JPY 7.121 < CHF 7.785
  • Weighted Ljung-Box tests: all series autocorrelated with ARCH/GARCH errors, so a model with a time-varying variance is appropriate
  • Highest unconditional correlation: EUR-CHF at 0.868

2 · Preparing the data and model

Returns and stationarity

  • Exchange rate levels are (near) random walks: non-stationary, as the paper's ERS tests confirm. Work with % log returns: \(y_t = 100\,[\ln x_t - \ln x_{t-1}]\)
  • Why the TVP-VAR wants stationary inputs: the connectedness measures come from the VMA (Wold) representation \(y_t=\sum_j B_{jt}\epsilon_{t-j}\), which only exists if the VAR is stable at each point in time; the GFEVD shares are then well defined at every horizon. Feeding in levels would push the roots to one and make the H-step variance shares meaningless
  • Log returns are also additive over time - which is exactly what lets us aggregate 5 daily returns into 1 weekly return without changing what the number means
  • Outliers: no trimming needed - the Kalman filter's gain automatically downweights observations that arrive with high forecast-error variance, one of the paper's four selling points

Run it yourself: the real data and its ADF tests

Daily EUR/GBP/JPY % log returns vs USD. Stationarity is the entry ticket to any VAR:

All three ADF statistics come out around -42 with p = 0.0000: returns are comfortably stationary, exactly as the paper finds with the ERS test on its monthly data. EUR-GBP correlate at 0.63, GBP-JPY at only 0.15. First run downloads Python into the browser (~15 MB).

Model choices: lag length, horizon, frequency

  • Lag length: the paper selects VAR(1) by BIC in both the rolling and TVP specifications (Table 3 notes). FX returns carry little linear memory, so one lag is standard; we check p = 2 in the robustness section
  • Forecast horizon: the paper uses H = 12 months for the GFEVD. We use H = 10 weeks as the baseline and vary it - you will see the shares stabilise long before that, so H is innocuous here
  • Aggregation: weekly returns (T = 1428) for the browser sims; the decimation keeps every episode (GFC, euro crisis, Brexit) visible while making 1400 Kalman steps + 1400 FEVDs instant in JavaScript
Scaling note. Returns are in percent. Connectedness shares are scale-free (each row of the GFEVD is normalised to 100%), so multiplying the data by any constant leaves every measure unchanged - but a sensible scale keeps the Kalman covariance matrices well conditioned.

Priors and forgetting factors: the model's only tuning knobs

  • Prior: the paper initialises the filter with the Primiceri (2005) / Del Negro and Primiceri (2015) prior - an OLS VAR on the first 60 months sets \(A_0\), \(\Sigma^A_0\) and \(\Sigma_0\). We do the same with the first 60 weekly observations
  • Forgetting factor \(\kappa_1\): discounts old information in the coefficients. Roughly, observation \(t-j\) gets weight \(\kappa_1^j\): with \(\kappa_1=0.99\) the effective memory is about \(1/(1-\kappa_1)=100\) observations; \(\kappa_1=1\) means constant coefficients (recursive OLS)
  • Decay factor \(\kappa_2\): the EWMA speed of the error covariance \(\Sigma_t\) - smaller means volatility updates faster. Benchmark values from Koop and Korobilis (2014): \(\kappa_1=0.99,\ \kappa_2=0.96\), kept constant (time-varying factors add computation but little forecast accuracy, Koop and Korobilis 2013)
  • Does the prior matter? The paper tries an uninformative prior, an informative prior and 500 random Minnesota priors: all converge to the same results after roughly 50 coefficient updates (Figures 9-10). The prior buys stability at the start, nothing more

3 · The method

The TVP-VAR(p) model (paper Equations 1-2)

$$y_t = A_t z_{t-1} + \epsilon_t, \qquad \epsilon_t\,|\,\Omega_{t-1}\sim N(0,\Sigma_t)$$ $$\operatorname{vec}(A_t)=\operatorname{vec}(A_{t-1})+\xi_t, \qquad \xi_t\,|\,\Omega_{t-1}\sim N(0,\Xi_t)$$
  • \(y_t\) is the m×1 return vector, \(z_{t-1}=(y_{t-1}',\dots,y_{t-p}')'\) stacks the lags
  • The coefficients \(A_t\) follow a random walk: they can drift every period, so a structural break is just a big step of \(\xi_t\)
  • Both \(\Sigma_t\) (error covariance) and \(\Xi_t\) (coefficient-innovation covariance) move over time - this is what "time-varying variance-covariance structure" means in the abstract

Compare with Week 7: the rolling-window approach re-estimates a constant-parameter VAR on a moving sample and hopes the window mimics parameter drift. Here drift is part of the model.

Kalman filter with forgetting factors (paper Section 2)

Prediction uses the forgetting factor \(\kappa_1\), which inflates the coefficient uncertainty instead of estimating the full \(\Xi_t\):

$$\Sigma^A_{t|t-1}=\tfrac{1}{\kappa_1}\Sigma^A_{t-1|t-1} \quad\Longleftrightarrow\quad \Xi_t=(\kappa_1^{-1}-1)\,\Sigma^A_{t-1|t-1}$$

Update moves the coefficients by the Kalman gain \(K_t\), and the error covariance follows an EWMA with decay \(\kappa_2\):

$$A_{t|t}=A_{t|t-1}+K_t\,(y_t-A_{t|t-1}z_{t-1}), \qquad \Sigma_{t|t}=\kappa_2\,\Sigma_{t-1|t-1}+(1-\kappa_2)\,\epsilon_{t|t}'\epsilon_{t|t}$$
  • What the gain does: small parameter uncertainty → stay near the prior state; large forecast errors with small \(\Sigma_t\) → adjust the coefficients strongly. This is the mechanism that makes the filter robust to outliers
  • No MCMC anywhere: the filter is deterministic and fast - each period is one set of matrix recursions, which is why we can run it live inside your browser

From filtered parameters to shocks: VMA and GIRF (Equations 3-13)

At each t, invert the (companion-form) VAR to its moving-average representation by the Wold theorem:

$$y_t=\sum_{j=0}^{\infty}B_{jt}\,\epsilon_{t-j},\qquad B_{jt}=J'M_t^{\,j}J$$

The generalised impulse response (Koop, Pesaran and Potter 1996; Pesaran and Shin 1998) is the difference between an H-step forecast with and without a one-standard-deviation shock to variable j:

$$\Psi_{j,t}(H)=\Sigma_{jj,t}^{-1/2}\,B_{H,t}\,\Sigma_t\,e_j$$
Why generalised? Cholesky-based decompositions (Diebold-Yilmaz 2009) depend on how you order the currencies. The GIRF integrates out the other shocks using their historical correlations, so the results are order-invariant - crucial when no currency has a natural "first mover" claim.

GFEVD and the total connectedness index (Equations 14-15)

The generalised forecast error variance decomposition asks: what share of the H-step forecast error variance of variable i is explained by shocks to variable j?

$$\tilde\phi_{ij,t}(H)=\frac{\sum_{h=0}^{H-1}\Psi^2_{ij,h,t}} {\sum_{j=1}^{m}\sum_{h=0}^{H-1}\Psi^2_{ij,h,t}} \qquad\text{(rows normalised: }\textstyle\sum_j\tilde\phi_{ij,t}=1\text{)}$$

Everything off the diagonal is spillover. The total connectedness index averages it:

$$C_t(H)=\frac{\sum_{i\ne j}\tilde\phi_{ij,t}(H)}{m}\times 100$$
  • \(C_t = 0\): each currency's forecast errors are entirely its own doing (no system)
  • \(C_t \to 100\): forecast errors are all imported - one tightly wired network

Directional, net, and pairwise measures (Equations 16-19)

TO others
$$C_{i\to j,t}=\sum_{j\ne i}\tilde\phi_{ji,t}\times 100$$
How much variable i's shocks explain of everyone else's forecast errors.
FROM others
$$C_{i\leftarrow j,t}=\sum_{j\ne i}\tilde\phi_{ij,t}\times 100$$
How much of i's own forecast errors are imported.
$$\text{NET: } C_{i,t}=C_{i\to j,t}-C_{i\leftarrow j,t} \qquad\text{NPDC: } NPDC_{ij}=(\tilde\phi_{ji,t}-\tilde\phi_{ij,t})\times 100$$
  • \(C_{i,t}>0\): variable i drives the network (net transmitter); \(C_{i,t}<0\): it is driven by the network (net receiver)
  • NPDC pins down each bilateral relationship: \(NPDC_{ij}>0\) means i dominates j

Rolling windows vs TVP-VAR: the paper's headline comparison

The Week 7 rolling-window VAR index overlaid on the TVP-VAR index (\(\kappa_1=0.99\)). Drag the window: short windows overreact, long windows flatten - and the grey gap on the left is the sample the rolling estimator throws away.
Reading the results. This reproduces the paper's Figures 5-7 logic on our data. At the default w = 104 weeks the two indices track each other (correlation 0.81) and their means are almost identical (28.1% vs 28.4%) - just as the paper finds the mean of all rolling TCIs matches the TVP-VAR. But push w to 52: the rolling index whipsaws (sd 8.5) and still loses a year of data. The TVP index needs no width choice and loses nothing: the paper's core claim, visible in one plot.

# the paper itself ships with the package: data(acg2020)
# teaching-grade numpy filter (full version: code/lecture08.py)
P = np.linalg.inv(Z0.T @ Z0)             # prior from OLS, first 60 obs
A = (P @ Z0.T @ Y0).T
S = np.cov((Y0 - Z0 @ A.T).T)
for t, (z, y) in enumerate(zip(Zlag, Y)):
    P = P / k1                           # forget old information
    Pz = P @ z; d = 1 + z @ Pz
    A = A + np.outer(y - A @ z, Pz / d)  # Kalman update
    P = P - np.outer(Pz, Pz) / d
    e = y - A @ z
    S = k2 * S + (1 - k2) * np.outer(e, e)
    tci[t] = gfevd_tci(A, S, H=10)       # GFEVD each period

  • The structure - who transmits, who receives, how the index breathes - reproduces almost perfectly on data the authors never used. That is what a robust method looks like

Using this in your dissertation

  • When TVP-VAR connectedness is the right tool: any network-of-variables question - volatility spillovers across markets, crypto contagion, sovereign yields, commodities and inflation - especially with short samples (no observations to burn on a window) or low-frequency data
  • Defaults to state and defend: \(\kappa_1=0.99\), \(\kappa_2=0.96\) (Koop-Korobilis benchmark; best forecaster in the paper's Tables 4-5), prior from the first ~60 obs, lag by BIC, H around 10-12, then show a sensitivity table like ours - referees will ask
  • Report all four layers: total index (system), TO/FROM (gross), NET (position), NPDC (bilateral). Each answers a different question; the paper's Figures 7-8 are the template
  • Software: ConnectednessApproach in R is the authors' own implementation and adds what our teaching filter omits: full Bayesian uncertainty bands (paper Figures 11-14), other estimators (QVAR, frequency-domain) and publication-ready plots
  • Honesty box: connectedness is a forecast-error-variance accounting, not causality; it inherits the VAR's linearity; and a falling index does not mean falling risk (March 2009 here)
